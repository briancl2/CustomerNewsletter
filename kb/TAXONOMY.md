# Taxonomy

## applies_to Tags

- `web`: GitHub.com web experiences.
- `vscode`: Visual Studio Code and extensions.
- `visual_studio`: Microsoft Visual Studio IDE.
- `jetbrains`: JetBrains IDE plugin surface.
- `xcode`: Xcode extension surface.
- `eclipse`: Eclipse plugin surface.
- `cli`: command-line tools and shell-first interfaces.
- `terminal`: terminal UI/chat surfaces (for example Windows Terminal Canary).
- `opencode`: OpenCode community surface.
- `mcp`: Model Context Protocol and MCP tooling.
- `agents`: agent orchestration, autonomy, or workflow frameworks.
- `skills`: skill package formats, conventions, or implementation guidance.
- `evals`: evaluation harnesses, benchmarks, and scorecards.
- `security`: safety, permissioning, and governance controls.
- `enterprise`: enterprise rollout, policy, and administration.
- `general`: cross-surface/general guidance.

## source_type Values

- `docs`: official documentation pages.
- `changelog`: chronological product change stream.
- `release-notes`: versioned release notes.
- `repo`: source repository.
- `marketplace`: marketplace listing/version pages.
- `rss`: RSS feed endpoint.
- `atom`: Atom feed endpoint.
- `api`: structured API endpoint.
- `standard`: protocol/specification standard.
- `practitioner-guide`: community/practitioner operational guidance.

## Stability Levels

- `high`: official source with durable URL/versioning behavior.
- `medium`: generally reliable but subject to reorganization, preview status, or frequent schema changes.
- `low`: early-stage, volatile, or weakly versioned source.

## Portability Labels

- `portable`: reusable across multiple tools/platforms with minimal adaptation.
- `platform-specific`: tied to one platform or vendor workflow.
- `deprecated`: historical or legacy guidance retained only for transition context.
